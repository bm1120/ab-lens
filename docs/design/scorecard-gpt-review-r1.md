# 교차 리뷰 — 3개 설계안 비판적 분석

---

## 1. 각 안의 핵심 강점과 결정적 약점

### [A] Claude 설계안

**강점**
- **"게이트"와 "채점" 분리** — 결격 차원(D1 측정가능성, D2 반증가능성)이 0이면 총점 무관 실격. 가설 품질이 평균이 아니라 최소 보장 문제라는 철학이 명확.
- **LLM을 이진 판정으로만 사용** — Y/N/부분(2/1/0) 구조화 호출로 재현성 확보. "값 생성"이 아닌 "판정"으로 한정한 설계가 탁월.
- **정체 감지 = "피드백 소진"** — 동일 미달 차원 2회 반복 시 즉시 종료. 점수 변동(±5 같은 임계)보다 안정적 신호.
- **피드백 메타 지시** — "통과한 차원은 건드리지 마라"로 sharpener가 전체를 흔드는 것 방지.

**약점**
- **통과 임계 78점이 높음** — 루프 과다 위험. "Acceptable with Caveat" 같은 중간 등급 없어 실무 유연성 부족.
- **자의적 임계값** — D5 가정 개수 Quick 2개/Deep 3개 차등 근거 불명확. mode 차이를 반영하려는 의도는 이해되나 휴리스틱.
- **메커니즘 D3이 룰+LLM 혼합인데 합산 로직 불명확** — "룰 10점 + LLM 10점 = 20점"이라 쓰고 코드는 `score_mechanism_structure` 0~10 반환. 나머지 10점 어떻게 더함?
- **D7 트레이드오프가 0/10 이분법** — 길이만 보고 품질 무시 (1개면 10점 vs 0개면 0점 → 조악).

---

### [B] GPT(내) 설계안

**강점**
- **적응적 통과 기준 (70점 Acceptable)** — 85점만 통과시키면 루프 과다. 70~84는 caveat 표시하고 진행 → 현실적.
- **"Information Gain" 철학** — "실험 후 무엇을 배울 수 있는가?"가 설계 중심. 반증조건을 독립 차원으로 강조(Popper).
- **차원별 하한선 → expander 재실행** — 구조적 결함(완결성/인과경로 붕괴)은 sharpener로 못 고친다는 인식. REFINE vs REDESIGN 구분 명확.
- **Plateau Detection** — 연속 2회 5점 이하 변동 시 조기 종료로 비용 절감.

**약점**
- **길이 기반 품질 판정** — "각 단계 15자 이상" 같은 형식주의. 길이≠품질 (A의 "실제 지표명 존재" 같은 의미론적 룰보다 조악).
- **LLM을 5점 척도로 사용** — A가 비판한 "재현성 낮음" 문제 그대로. temperature=0이어도 ±1점 오차 인정 → 노이즈.
- **7개 차원 다소 많음** — D1~D7 중 일부 중복 (측정 체계 정렬 vs 구조적 완결성). 계산 복잡도↑.
- **피드백 템플릿 장황** — A의 "통과한 차원 건드리지 마라" 같은 메타 지시 없음. sharpener가 모든 차원을 다시 건드릴 위험.

---

### [C] Gemini 설계안

**강점**
- **구조적 완전성을 필수 게이트로** — A의 결격 차원 철학과 유사. "총점 무관 게이트" 명시.
- **Rule 70% + LLM 30% 비율 명시** — 설계 의도가 정량적으로 명확 (A/B는 비율 암묵적).
- **동일 Rule 차원 2회 연속 실패 시 즉시 중단** — sharpener가 스키마를 못 맞추는 상태 조기 감지.
- **피드백 JSON 구조** — `failed_dimensions` 배열이 A의 딕셔너리보다 체계적.

**약점**
- **인과 논리 타당성 차원에 30점(전체 30%)을 LLM에 할당** — 재현성 최악. A/B는 LLM 비중 최소화했는데 역행.
- **메커니즘 구체성이 "화살표 2개"만 검증** — 구조만 보고 인과 내용 무시. A의 LLM 타당성 판정이나 B의 길이 기준보다도 표면적.
- **대안적 사고(D4) 10점이 낮음** — A의 교란 인지(D6), B의 대안 탐색(D5)과 겹침. 차원 설계 중복.
- **통과 임계 85점 단일** — A의 78점보다 높고, B의 70점 Acceptable 개념 없음 → 루프 과다.

## 4. 미해결 쟁점

### 쟁점 1: 메커니즘 평가의 룰vs타당성 경계

- **A**: 룰(3-segment 구조 0~10) + LLM(인과 타당성 Y/P/N → 10/5/0) = 합산 20점  
  → **합산 로직 불명확** (코드는 `score_mechanism_structure` 0~10만 반환, 나머지 LLM 10점 어떻게 더함?)
- **B**: 룰만(화살표 2개 + 각 15자) → 지나치게 형식적
- **C**: 룰만(화살표 2개) → 너무 표면적

**미해결**: 구조(형식)와 인과 타당성(내용)을 어떻게 독립적으로 채점하고 합산?  
**제안**: `메커니즘 차원 = 구조(룰 0~10) + LLM(Y/P/N → +10/+5/+0) = 최대 20점` 명시. LLM이 N이어도 구조 점수는 살림.

---

### 쟁점 2: 통과 임계 70 vs 78 vs 85

- **A**: 78점 + 결격 통과 → PASS (단일 게이트)
- **B**: 70~84(Acceptable, caveat 표시) / 85+(Ready, 무조건 통과)
- **C**: 85점 단일

**미해결**: "충분히 좋은" 지점을 어디로 잡을 것인가?  
**제안**: B의 2단계 채택하되, **Acceptable은 결격 차원 필수 통과 + 총점 70** 조건 추가 (측정·반증 보장 없으면 70점도 무의미).

---

### 쟁점 3: 정체 감지 — 점수 기반 vs 차원 기반

- **A**: 동일 미달 차원 집합 2회 반복
- **B**: 점수 Δ<5 연속 2회
- **C**: 점수 Δ≤5 OR 동일 Rule 차원 2회

**미해결**: 점수 노이즈(LLM 변동)가 큰 환경에서 어느 지표가 더 안정적?  
**나의 판단**: **A의 차원 기반이 더 신뢰 가능**. 점수는 LLM이 흔들지만 "D3 메커니즘이 2회 연속 미달"은 명확한 모델 한계 신호.  
**BUT**: C의 "동일 Rule 차원 2회"도 합리적 (LLM 변동 배제). → **통합 제안**: "동일 미달 차원(Rule+LLM 포함) 집합 2회 반복 OR 동일 Rule 차원만 2회 반복" 둘 다 걸림.

---

### 쟁점 4: D5(가정 노출) 임계 — Quick 2개 vs Deep 3개?

- **A**: mode별 차등 (Quick 2 / Deep 3)  
  근거: "Deep은 2라운드 sharpener라 발산 깊이 차이"
- **B/C**: 명시 없음

**미해결**: mode별 차등이 필요한가? 아니면 통일 (예: 3개)?  
**나의 판단**: **차등 삭제, 통일 3개**. 이유:
1. Quick도 가정 3개 못 뽑으면 실험 설계 부실.
2. mode별 차등은 임계값 관리 복잡도↑, 실질적 품질 차이는 미미.
3. "발산 깊이"는 rejected_alternatives나 confounder 개수로 이미 반영됨.

---

### 쟁점 5: 트레이드오프 평가 — 독립 차원 vs 측정/리스크 통합?

- **A**: D7 독립 차원 10점, 단순 길이 (1개 = 10점 / 0개 = 0점)
- **B**: D4 측정 체계 정렬의 일부 4점
- **C**: D3 리스크 인지에 통합 (tradeoff + confounder 합산 15점)

**미해결**: 트레이드오프 예측을 독립 차원으로 볼지, 리스크/측정에 통합할지?  
**나의 판단**: **독립 차원 유지 [A/B]**. 이유:
- 트레이드오프는 "가드레일 지표 예측"으로서 실험 설계의 핵심 (단순히 리스크나 측정의 부속물 아님).
- 단, A의 0/10 이분법은 조악 → **"1개 = 7점 / 2개 이상 = 10점 / 0개 = 0점"** 세분화.

---

### 쟁점 6: expander 재실행 조건 — 총점 vs 구조적 결함?

- **A**: 총점 <60 OR 결격 차원 미달 → REDESIGN
- **B**: 완결성 <16 OR 인과경로 <10 → Structural Failure
- **C**: 총점 <60 → Back to Drawing Board

**미해결**: "sharpener로 못 고친다"의 기준을 총점으로 볼지, 특정 차원 하한으로 볼지?  
**나의 판단**: **B의 차원별 하한 채택**. 이유:
- 총점 60점이어도 결격 차원(측정·반증) 통과 + 나머지 고르면 → 보강 가능.
- 반대로 총점 65점이어도 인과경로 0점(메커니즘 비어있음)이면 → 구조적 결함, expander 재실행 필요.
- **통합 조건**: `(D1 < 16 OR D3 < 10) AND 총점 < 70` → expander (1회만).

### 최종 미해결

1. **LLM 이진 판정의 실제 재현성** — temperature=0 + enum 강제해도 Y/N 경계에서 흔들리는 케이스 존재. 2회 채점해서 일치율 측정 필요?
2. **룰베이스 임계값의 도메인 특수성** — ab-lens는 A/B 테스트 맥락인데, 다른 실험 유형(예: 탐색적 분석)에도 동일 기준 적용 가능? 아니면 context별 가중치 조정?
3. **피드백 소진 vs 모델 전환** — 같은 차원에서 2회 막혔을 때, 루프 종료가 맞나 **다른 모델로 재시도**가 맞나? (예: Claude가 못 고친 D3을 GPT에게 넘김)
