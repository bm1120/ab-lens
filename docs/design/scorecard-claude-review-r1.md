# ab-lens HQS 교차 리뷰 (참가자: Claude / 설계안 A 저자)

먼저 메타 관찰 하나. **세 안이 독립 설계인데도 골격이 거의 수렴했다** — 7±2 차원, 룰 다수 + LLM 소수, 결격 게이트, 정체감지, expander 되돌림. 이건 설계 공간이 좁다는 신호이자, **수렴한 부분은 안전하고 갈라진 부분이 진짜 쟁점**이라는 뜻이다. 아래는 갈라진 곳만 집중한다.

## 1. 각 안의 핵심 강점 / 결정적 약점

### [A] 내 안 — 객관적 자기비판부터

**강점**
- LLM을 **점수 생성이 아닌 Y/P/N 이진 판정**으로만 쓴 것 (D2/D3). 재현성 관점에서 셋 중 가장 방어적.
- 정체감지를 점수 Δ가 아니라 **"동일 미달 차원 집합 반복"**으로 본 것. 점수는 LLM 노이즈로 ±5 흔들려도 *어디서 막히는가*는 안정 신호다 — 이건 B/C의 Δ방식보다 명백히 우월하다(근거는 3번에서).

**결정적 약점 (스스로 깐다)**
- **임계값 인플레이션.** 78점 + D1==20 + D2통과 + D3≥12 — 게이트를 4중으로 걸어놓고 또 78을 요구한다. 결격 게이트가 이미 "측정·반증 보장"을 강제하는데 78은 **이중 처벌**이다. 게이트 통과 후엔 임계를 낮춰야 일관된다. 내 78은 정당화가 약하다.
- **D5 가정 개수 임계(2/3)가 길이 휴리스틱으로 회귀.** 내가 B를 "개수 세기"라고 깔 자격이 없다. D5는 내 설계에서 가장 약한 고리다.
- **`mode`가 스코어카드에 이미 전달된다는 가정**에 결격 로직이 의존. 전달 안 되면 D5 임계가 깨진다.

### [B] GPT 안

**강점**
- **반증조건을 독립 차원으로 승격**(D3). Popper 프레이밍은 셋 중 가장 명시적이다. "언제 틀렸다고 할 것인가"를 차원으로 박은 건 옳다.
- **차원별 하한선 + expander 트리거를 명문화**(구조완결<16 또는 인과<10 → expander). A는 결격을 D1·D2에만 걸었는데, B는 "구조 결함은 sharpener로 복구 불가"라는 **인과적 구분**을 더 명확히 했다. 이건 내 안보다 낫다.

**결정적 약점**
- **"단계당 15자 이상"(D2 인과경로)은 게이밍 가능 최악의 룰.** 글자 수는 의미와 무관하다. "사용자가 더 좋아져서 더 많이 클릭함" = 17자, 통과. 의미는 0. **길이는 품질의 프록시가 아니다** — B 설계 전체에서 가장 위험한 결정.
- **80/20 비율을 선언만 하고 강제 안 함.** LLM은 D3(부분)+D7(5점)뿐인데 "20%"라는 숫자가 어디서 나오는지 불명. 가중치 합으로 20이 안 된다.
- **토큰 예산 50k 상한**은 좋은 발상이나 **턴 제한과 중복**. 턴이 이미 막는데 토큰은 거의 안 걸린다 — 죽은 정책.

### [C] Gemini 안

**강점**
- **가장 단순하다 (5차원).** 7차원 두 안보다 구현·디버깅·설명이 쉽다. 과적합 위험이 가장 낮다. MVP로는 이게 맞을 수 있다.
- **인과 타당성에 30% 몰빵.** "가설 품질의 핵심은 인과논리"라는 철학을 가중치로 정직하게 표현했다. B의 25%보다 더 공격적이고 일관적.

**결정적 약점**
- **LLM에 30점을 직접 생성시킨다 (`logical_score 0~30`).** 이건 셋 중 **재현성 최악**. 30점 연속 스케일은 호출마다 ±5~8 흔들린다 — 게이트가 이 위에 서 있으면 통과/미달이 동전던지기가 된다. A가 이걸 피하려고 Y/P/N으로 잘게 쪼갠 것과 정면충돌.
- **리스크 인지(D3)에서 tradeoff AND confounder를 한 차원에 묶음.** 둘은 다른 실패모드(부작용 예측 vs 인과위협)인데 AND로 묶으면 하나만 빠져도 차원 전체가 죽어 피드백 해상도가 떨어진다.
- **정체감지가 Δ≤5 + 동일차원반복 둘 다 OR로 종료** → 너무 쉽게 멈춘다. 1턴 만에 4점 오르고 멈추면 개선 여지를 버린다.

| # | 채택 아이디어 | 출처 | 이유 |
|---|---|---|---|
| ① | **결격(disqualifier) 게이트 — 측정·반증은 총점 무관 차단** | A (+ C도 구조완전성에 동일) | 가설품질은 평균이 아니라 최소보장 문제. 세 안 중 둘이 독립 수렴 → 강한 신호 |
| ② | **반증조건을 독립 차원으로** | B (D3) | Popper 프레이밍. A의 D2보다 B가 더 정면으로 다룸 |
| ③ | **구조결함 vs 정제가능결함 구분 → 미달 종류별 라우팅**(sharpener vs expander) | B (하한선+트리거) | 결격을 "차단"으로만 본 A보다 "어디로 되돌릴지"까지 정한 B가 우월 |
| ④ | **LLM은 연속점수 금지, Y/P/N 이진 rubric** | A | C의 0~30 직접생성 재현성 붕괴를 막는 핵심. 게이트가 LLM 위에 설 거면 필수 |
| ⑤ | **정체감지 = 동일 미달 차원 집합 반복** | A (+ C도 "동일차원 반복실패" 보유) | 점수 Δ보다 안정적. A·C 독립 수렴 |
| ⑥ | **인과 타당성 최고 가중치** | C(30) / B(25) | 가설품질의 척추. 단 *판정은 이진으로*(④와 결합) |
| ⑦ | **피드백 = "지적된 필드만 수정, 통과 차원은 건드리지 마라" 제약** | A·C 공통 | 단조수렴 유도. 매턴 전체 흔들림 방지 |
| ⑧ | **best-so-far 반환 (마지막이 역행했을 수 있음)** | A·C 공통 | 종료 시 argmax(total) 채택 |

## 3. 버려야 할 것 / 충돌하는 설계 결정과 내 판단

**버린다**
- **B의 "단계당 15자 이상"** — 길이 게이밍. 구조(화살표 개수)는 룰로 보되, 의미 충족은 ④ LLM Y/P/N으로 넘긴다.
- **C의 `logical_score 0~30` 직접생성** — 재현성 붕괴. C의 30% *가중치 철학*은 살리되 *판정 방식*은 A의 이진으로 교체.
- **B의 토큰 50k 상한** — 턴 제한과 중복. 죽은 정책.
- **내(A) D5 가정 개수 2/3 임계** — 길이 휴리스틱 회귀. 차라리 D5를 LLM Y/P/N "숨은 전제가 실험을 위협할 수준으로 드러났나"로 교체하거나 가중치 삭감.

**정면충돌 → 내 판단**

| 충돌 축 | A | B | C | 내 결론 |
|---|---|---|---|---|
| LLM 판정 방식 | Y/P/N 이진 | 5점+부분 | 0~30 연속 | **A 채택.** 게이트가 LLM 위에 서면 연속점수는 동전던지기 |
| 통과 임계 | 78(높음) | 70 acceptable | 85 | **B의 2단계(Acceptable+caveat) 채택**, 단 임계는 결격 게이트 이후라 낮춰도 됨. 내 78은 이중처벌 |
| 정체감지 | 동일차원집합 | Δ≤5 | 둘다 OR | **A의 동일차원**을 주신호, Δ는 보조. C처럼 OR로 쉽게 멈추지 말 것 |
| 차원 수 | 7 | 7 | 5 | **6으로 수렴 권장.** C의 단순성 + A/B의 반증·트레이드오프 분리 절충 |

## 4. 미해결 쟁점 (정직하게 — 안 풀렸다)

1. **룰 vs LLM 비중의 "정직한" 숫자.** 셋 다 "룰 80 / LLM 20"류를 말하지만 *가중치 합으로 강제한 안이 없다*(B는 선언만, C는 LLM에 30% 줘서 모순). → 미해결: LLM 가중 상한을 25%로 박을지, 아니면 LLM은 게이트(Y/N)로만 쓰고 점수엔 0% 기여시킬지. 후자가 더 방어적이지만 인과 타당성을 점수에서 빼는 대가가 있다.

2. **통과 임계값의 절대 근거 없음.** 78/85/70 — 셋 다 휴리스틱이고 데이터가 없다. → **실험 데이터 없이는 임계 정당화 불가.** 합의 가능한 건 "결격 게이트 + 상대 등급(2단계)"뿐. 절대 임계는 **실제 가설 20~30개에 셋이 채점해보고 캘리브레이션**해야 한다(미수행).

3. **정체감지 임계의 트레이드오프.** "동일 차원 2연속"이 정답 같지만, *2턴 만에 종료*면 Deep 모드에서 너무 짧다. 동일차원 + Δ를 AND로 걸지 OR로 걸지 미합의. C는 OR(공격적), 나는 단일신호 — **둘 다 근거는 직관뿐.**

4. **인과 타당성을 게이트로 vs 점수로.** C는 점수(30), B는 점수(25), A는 PASS조건(D3≥12)으로 반쯤 게이트화. → 메커니즘 비면 사후해석이 깨지니 **게이트여야 한다는 게 내 직관**이지만, 초기 발산 단계 가설을 너무 일찍 죽일 위험(false negative) 미검증.

5. **`HypothesisOutput` 필드 신뢰성.** 세 안 모두 `confounder_candidates`, `rejected_alternatives` 등이 *진짜로 채워진다*고 가정. 빈 배열이 흔하면 룰 차원 절반이 0점 → 전체가 REDESIGN으로 쏠린다. **상류 에이전트 출력 분포를 안 봤다 (셋 다 미검증).**

## 한 줄 결론

세 안의 수렴부(결격 게이트·룰다수·정체감지·best-so-far)는 즉시 통합 가능하고, **진짜 싸움은 딱 두 곳** — (1) LLM을 점수에 넣을지 게이트로만 쓸지, (2) 절대 임계를 데이터 없이 정할지다. 내 입장: **LLM은 이진 게이트로만, 절대 임계는 캘리브레이션 전까지 봉인하고 결격+상대등급으로만 출발.** 내 원안의 78점과 D5는 철회한다.
